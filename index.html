Analytics & Modeling

Global Retail Executive Dashboard

Standardizing multi-currency performance for global stakeholders.

The Thought Process

"How do we compare a €50,000 sale in Berlin to a $55,000 sale in New York without losing the local context?"

The Reason

Executive teams were struggling with disparate CSV reports where exchange rates were applied inconsistently. We needed a single source of truth that allowed for dynamic currency switching.

Execution

I engineered a Star Schema consisting of 1 Fact table (Sales) and 4 Dimension tables (Geography, Product, Time, Currency). Using DAX, I created a measures library that normalized all transactions to a base currency (USD) while maintaining a slicer for local viewing.

The Difficulty

Handling "SCD Type 2" (Slowly Changing Dimensions) was the hurdle. When a product changed category mid-year, the historical sales needed to remain in the old category while new sales went to the new one. I solved this by mapping transactions to a Surrogate Key in the product dimension rather than the Product ID.

Cloud Architecture

AWS WhatsApp Intake Pipeline

Turning mobile messages into structured database entries.

The Reason

Nexum Group needed a way to intake legal claim data from users 24/7 without paying for a full-time support staff or maintaining expensive, idle servers.

Execution

I built a Serverless Architecture using AWS Lambda and API Gateway. Messages are received via a Twilio webhook, processed by a Python-based Lambda function, and stored in an RDS instance. This ensures the company only pays when a message is actually sent.

The Difficulty

Concurrency and cold starts. Initially, the first message of the day had a 2-second delay as the Lambda "woke up." I optimized this by slimming the deployment package (removing unused libraries) and implementing a Provisioned Concurrency plan for peak hours.

Python & ETL

Financial Data Automation

Replacing 4 hours of manual labor with 45 seconds of code.

The Thought Process

Manual Excel work isn't just slow; it’s an invitation for human error. If a process is repeatable, it must be automated.

Execution

Developed a Python ETL script using **Pandas** and **Openpyxl**. The script ingests raw, messy CSV exports from Shopify and Amazon, applies complex cleaning logic (regex for SKU standardization), and outputs a formatted executive summary.

The Difficulty

Data drift. The third-party platforms would occasionally change their header names without notice, breaking the script. I implemented a **Schema Validation** layer using Pydantic to catch these errors early and alert the user which column was missing.

Streaming Fraud Monitor

Detecting fraudulent legal claims requires analyzing data as it arrives, not 24 hours later.

Execution: Using Apache Spark for stream processing and Snowflake as the data warehouse.

Difficulty: Latency vs. Accuracy. Filtering out "false positives" in a live stream is computationally expensive. I am testing a "Two-Pass" system where Spark does a light filter and Snowflake handles the deep check.
